## Overview

Данный проект демонстрирует реализацию инструмента для сохранения и восстановления прав доступа (включая ACL) к файлам и каталогам. Программа упаковывает содержимое и метаданные прав доступа в единый ZIP-архив, что позволяет переносить файлы между компьютерами с сохранением всех настроек безопасности.

## Projects

В состав лабораторной работы входит один программный компонент:

### Приложение (`save_permissions.py`)

Программа реализована на языке Python и поддерживает два режима работы: командную строку (CLI) и графический интерфейс (GUI). Для сохранения прав доступа используются нативные инструменты ОС: PowerShell + SDDL на Windows и `getfacl`/`setfacl` на Linux.

**Особенности реализации:**
- Поддержка Windows и Linux (кроссплатформенность).
- Сохранение и восстановление расширенных ACL, а не только базовых флагов.
- Работа как с отдельными файлами, так и с целыми каталогами (рекурсивный обход).
- Упаковка данных в ZIP-архив с файлом метаданных `metadata.json`.
- Графический интерфейс на базе библиотеки `tkinter`.
- Проверка совместимости ОС при восстановлении прав.

## How to build

### Требования к окружению

| Компонент | Минимальная версия |
|-----------|-------------------|
| Python | 3.6 и выше |
| pip | 20.0 и выше |
| PowerShell | 5.0 (Windows) |
| acl | 2.2.53 (Linux) |

### Установка зависимостей

Стандартные библиотеки Python не требуют дополнительной установки. Для Linux необходимо установить пакет `acl`:

```bash
# Ubuntu/Debian
sudo apt install acl

# CentOS/RHEL/Fedora
sudo yum install acl
```

### Запуск программы

#### Графический интерфейс (GUI)

```bash
python save_permissions.py
```

#### Командная строка (CLI)

**Сериализация файла:**
```bash
python save_permissions.py --serialize --input testfile.txt --output backup.zip
```

**Сериализация каталога:**
```bash
python save_permissions.py --serialize --input ./my_folder --output backup.zip
```

**Десериализация:**
```bash
python save_permissions.py --deserialize --input backup.zip --output ./restored
```

## How it works

### Формат хранения

Выходной файл представляет собой ZIP-архив со следующей структурой:

```
backup.zip
├── metadata.json          # Метаданные с правами доступа
├── file1.txt              # Исходный файл
├── file2.txt              # Исходный файл
└── subfolder/             # Вложенная папка
    └── file3.txt
```

### Структура metadata.json

```json
{
    "os": "Windows",
    "permissions": {
        "file1.txt": "O:S-1-5-21-...G:S-1-1-0...",
        "subfolder": "O:S-1-5-21-...G:S-1-1-0...",
        "subfolder/file3.txt": "O:S-1-5-21-...G:S-1-1-0..."
    }
}
```

- `os` — операционная система, на которой был создан архив.
- `permissions` — словарь, где ключ — относительный путь, значение — ACL в формате SDDL (Windows) или вывод `getfacl -c` (Linux).

### Алгоритм работы

#### Сериализация
1. Определение типа входного пути (файл или каталог).
2. Получение ACL для каждого файла/папки через системные утилиты.
3. Упаковка файлов в ZIP-архив.
4. Добавление файла `metadata.json` с сохранёнными правами.

#### Десериализация
1. Чтение ZIP-архива и извлечение `metadata.json`.
2. Проверка совместимости ОС (предупреждение при несовпадении).
3. Распаковка всех файлов в целевой каталог.
4. Применение ACL к каждому файлу/папке при совпадении ОС.

## Additional Notes

### Особенности реализации

1. **Выбор языка:** Python выбран для реализации в соответствии с требованием о возможности выбора языка программирования. Это позволило обеспечить кроссплатформенность и использовать нативные системные утилиты для работы с ACL.

2. **Хранение прав доступа:** На Windows используется SDDL (Security Descriptor Definition Language) — строковое представление дескриптора безопасности, которое полностью описывает все права доступа. На Linux используется вывод `getfacl -c`, который также является переносимым текстовым форматом.

3. **Обработка ошибок:** Программа корректно обрабатывает все возможные исключения:
   - `FileNotFoundError` — при отсутствии входного файла/каталога.
   - `PermissionError` — при недостатке прав для чтения/записи.
   - `ValueError` — при отсутствии metadata.json в архиве.

4. **GUI особенности:** В интерфейсе предусмотрены отдельные кнопки для выбора файла и каталога, что исключает путаницу при сериализации разных типов объектов.

5. **Тестирование:** Скрипт `run_tests.py` автоматически проверяет корректность сохранения сложных ACL, включая запрет записи на Windows и ACL для nobody на Linux.
